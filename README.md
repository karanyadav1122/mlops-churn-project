#  Customer Churn Prediction MLOps System

##  Overview

This project is a production-style end-to-end MLOps system for customer churn prediction, combining batch training and real-time streaming inference.

It demonstrates how to design and deploy a scalable ML pipeline using:

- PySpark for data processing and model training
- Kafka for real-time streaming
- Spark Structured Streaming with Bronze-Silver-Gold architecture
- MLflow for experiment tracking and model management
- FastAPI for inference serving
- Docker for containerized deployment

---

##  Architecture

### Batch Pipeline (Training)

Raw Data → Feature Engineering → Model Training → MLflow Tracking → Model Registry

### Streaming Pipeline (Real-Time Inference)

Producer → Kafka (`churn_input`)  
↓  
Consumer → Model Serving (FastAPI / MLflow)  
↓  
Kafka (`churn_output`)

### Streaming Data Processing (Feature Engineering)

Kafka → Spark Structured Streaming  
↓  
Bronze → Silver → Gold  
↓  
Kafka (`churn_features`)

##  How to Run

### 1. Clone the repository

    git clone <your-repo-url>
    cd mlops-churn-project

### 2. Create and activate virtual environment

    python -m venv venv
    source venv/bin/activate

### 3. Install dependencies

    pip install -r requirements.txt

### 4. Start Docker services

    docker compose up -d

### 5. Verify running services

    docker compose ps

### 6. Start Spark streaming jobs (if running manually)

    python spark/bronze_stream.py
    python spark/silver_transform.py
    python spark/gold_stream.py

### 7. Run Kafka producer

    python streaming/producer.py

### 8. Run Kafka consumer

    python streaming/consumer.py

### 9. Check prediction output

    docker compose exec kafka kafka-console-consumer \
      --bootstrap-server kafka:9092 \
      --topic churn_output \
      --from-beginning
